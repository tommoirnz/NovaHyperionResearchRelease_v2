Part 2: ATmega Quadrature Decoding

State Machine Approach

Define the 2-bit state:

Sk={Ak,Bk}∈{00,01,10,11}

The transition determines direction:

Δ=Sk⊕Sk−1

The canonical lookup table maps previous→current state to increment:

Valid forward transitions: 00→10→11→01→00

Valid reverse transitions: 00→01→11→10→00

ATmega Hardware Options

Option A — Pin Change Interrupts (PCINT):

Trigger on any edge of A or B⇒ISR reads both pins, updates state

Latency constraint:

fencoder\_max<fCPUCISR\_cycles⋅4

Option B — External Interrupts INT0/INT1:

Trigger on rising edge of A only⇒read B to determine direction

Gives ×2 decoding (half resolution of ×4).

Option C — Timer Input Capture:

Captures timestamp Tn on each edge:

Tpulse=Tn−Tn−1

Best for velocity measurement precision.

Part 3: PWM Theory for Motor Control

Timer/Counter PWM on ATmega

The ATmega Timer generates PWM via compare match. For Fast PWM mode:

fPWM=fCPUN⋅(TOP+1)

Where N is the prescaler value and TOP is the counter ceiling (255 for 8-bit, 65535 for 16-bit).

The duty cycle:

D=OCRTOP+1∈[0,1]

The average voltage delivered to the motor:

V¯motor=D⋅Vsupply

Phase-Correct PWM (preferred for motor control)

Counter counts up then down (dual-slope):

fPWM=fCPU2⋅N⋅TOP

Lower frequency but symmetrical switching — reduces torque ripple.
