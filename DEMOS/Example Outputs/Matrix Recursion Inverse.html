Soleymani (2012) — Seventh-Order Matrix Inversion Algorithm

Paper: "A Rapid Numerical Algorithm to Compute Matrix Inversion" Author: F. Soleymani, Ferdowsi University of Mashhad, Iran

The Core Problem

Given an invertible matrix A, find A⁻¹ iteratively — especially useful when A is large, sparse, or ill-conditioned (where direct methods like Gaussian elimination become numerically unreliable).

The Algorithm Family (Historical Context)

Each generation improved the convergence order — meaning how fast the error shrinks per iteration:

The Seventh-Order Method — Equation (3.1)

Vn+1=116Vn(120I+AVn(−393I+AVn(735I+AVn(−861I+AVn(651I+AVn(−315I+AVn(93I+AVn(−15I+AVn))))))))

This is a nested (Horner-style) matrix polynomial in AVn, evaluated from the inside out.

Why "Seventh-Order"?

Define the error matrix at step n:

En=I−AVn

Seventh-order convergence means:

‖En+1‖≤C⋅‖En‖7

So if your current error is 10−2, the next iteration gives error ≈10−14. That's one iteration doing the work of many.

Convergence Condition

The iteration converges if the initial guess V0 satisfies:

‖I−AV0‖<1

A standard choice is:

V0=α‖A‖1⋅‖A‖∞AT

where α∈(0,2) — this guarantees the spectral radius condition is met.

The Coefficients — Where Do They Come From?

The coefficients {120,−393,735,−861,651,−315,93,−15,1} are not arbitrary. They are derived so that the Taylor expansion of En+1 in terms of En eliminates all terms up to order 6, leaving:

En+1=En7⋅(bounded matrix)

This is the algebraic fingerprint of seventh-order convergence.

In Plain English

Start with a rough guess V0 for A−1. Apply the formula. Each application makes your approximation seven times more accurate (in the exponent). After just 3–4 iterations, you're at machine precision — even for matrices that make Gaussian elimination sweat. 🎯

The elegance here is that it's entirely built from matrix multiplications — no divisions, no decompositions, just multiply and nest. That makes it both theoretically clean and computationally efficient on modern hardware.
