What NovaCouncil Actually Is

Five specialised agents — Analyst, Creative, Critic, Strategist, Empath — each with a distinct cognitive role and output format. They're not arbitrary personas; they represent modes that a single-pass LLM call tends to collapse together. A vanilla response optimises for plausibility and fluency. The Council forces independent passes that prioritise accuracy, novelty, fault-finding, long-term thinking, and emotional tone separately, before synthesis.

The wiring is clean:

  • Task-type routing maps question categories (research, creative, math, code, social) to the most relevant agent subset
  • Parallel execution via ThreadPoolExecutor — agents run concurrently, not sequentially, so latency doesn't stack
  • Sequential mode is also available, where each agent sees prior outputs — useful for iterative refinement
  • Deliberation cache with a 5-minute TTL avoids redundant calls on repeated queries
  • Synthesis step produces a final Nova response that integrates all perspectives without exposing the internal scaffolding

Where It's Genuinely Better

  • Complex decisions — architectural, strategic, or philosophical questions where a single agent tends toward confident but shallow answers
  • Creative work — the Analyst would normally suppress lateral ideas; separating Creative and Analyst lets both breathe
  • Risk assessment — the Critic surfaces failure modes that a single optimistic pass glosses over
  • Research synthesis — Analyst + Strategist + Critic is a strong combination for evaluating evidence and implications

Where It's Pure Overhead

  • Factual lookups — weather, dates, simple facts don't need a committee
  • Simple code tasks — four agents debating a loop is theatre
  • Conversational exchanges — latency becomes actively annoying when you just want a quick answer

The task-type router partially addresses this — default only invokes Analyst + Empath, which is lightweight. But the routing decision still needs to be sharp.

The Weak Link

The synthesis step. The Council is only better than a single well-prompted call if the synthesiser is genuinely integrating perspectives — not just appending them with headers. The current synthesis prompt is reasonable, but if the underlying generate() call doesn't have enough context or instruction weight, you can end up with a response that's less coherent than a single focused call, at 3–5× the cost and latency.

Verdict

Better for hard problems. Worse for routine ones. The architecture is sound — parallel agents, task-type routing, caching, and a synthesis layer are all the right ideas. The bones are good.

The missing piece is a pre-deliberation classifier that decides: does this question actually warrant the full Council, or should it bypass to a single-agent fast path? Without that, you're firing a committee at every question — including the ones that just need a quick answer.

Wire that classifier in, and the Council becomes a genuine upgrade. As it stands, it's a powerful tool that's occasionally used on nails that don't need a sledgehammer.
